# Contributing to FileKakari

FileKakari への貢献をご検討いただきありがとうございます。プロジェクトへ協力・参加いただく際は、以下のガイドラインに従ってください。

## 開発・動作環境
* **前提環境**: Windows OS、.NET 10 (WPF)
* 動作確認およびビルド確認は、必ず事前に Windows/PowerShell 上で行ってください。

## 変更・提案のフロー
1. **問題の報告・提案**:
   * 不具合報告や機能の提案は、GitHub の Issues にて受け付けています。
2. **事前相談**:
   * 大規模なリファクタリングや機能追加を行う場合は、実装を開始する前にあらかじめ Issue を作成し、開発方針についてご相談ください。
3. **Pull Request のルール**:
   * PR は基本的に「1つの目的（1つの責務）」に絞って作成してください。
   * 送信前に、ローカルで `dotnet build` が正常に通ることを必ず確認してください。

## コミットメッセージについて
* 本プロジェクトの内部メンバーはコミットメッセージの先頭に `Ag:` / `Cd:` などの接頭辞を付与していますが、**外部コントリビューターの方にはこの接頭辞規約は要求しません**。任意の明確なメッセージでコミットを作成していただいて構いません。
