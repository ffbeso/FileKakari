# FileKakari 操作ガイド / Keyboard Shortcuts

FileKakari の基本操作とキーボードショートカットをまとめます。

FileKakari は、マウス操作だけでなく、キーボードだけでも基本的なファイル操作・タブ操作・Workspace操作を行えることを目指します。

---

## 1. 基本方針

* Windows Explorer で一般的なショートカットは、できるだけ同じ意味で使う
* FileKakari 独自の Workspace / ペイン / サブタブ操作は、専用ショートカットを割り当てる
* TextBox、パスバー、フィルター、リネーム中の標準入力操作を奪わない
* 通常フォルダ、単一ペインWorkspace、複数ペインWorkspaceでできるだけ同じ操作体系にする
* 入力イベントは薄く保ち、実処理は共通メソッドへ集約する

---

## 2. ナビゲーション操作

| 操作       | ショートカット     | 備考                          |
| -------- | ----------- | --------------------------- |
| 戻る       | `Alt+Left`  | Windows標準。ただし外部WM等に奪われる場合あり |
| 進む       | `Alt+Right` | Windows標準。ただし外部WM等に奪われる場合あり |
| 戻る       | `Ctrl+[`    | 代替ショートカット                   |
| 進む       | `Ctrl+]`    | 代替ショートカット                   |
| 親フォルダへ移動 | `Alt+Up`    | Windows Explorer相当          |
| 親フォルダへ移動 | `Backspace` | ファイル一覧にフォーカスがある場合のみ         |
| 更新       | `F5`        | 現在のフォルダ一覧を再読み込み             |

### 注意

`Alt+Left` / `Alt+Right` は、Komorebi などのタイル型ウィンドウマネージャや常駐アプリにグローバルショートカットとして奪われる場合があります。

その場合は代替として以下を使います。

```text
Ctrl+[   戻る
Ctrl+]   進む
```

## 3. パスバー操作

| 操作         | ショートカット  | 備考                |
| ---------- | -------- | ----------------- |
| パスバーへフォーカス | `Ctrl+L` | 現在パスを選択して編集       |
| パスバーへフォーカス | `Alt+D`  | Explorer / ブラウザ相当 |
| 入力パスへ移動    | `Enter`  | パスバー編集中           |
| 編集キャンセル    | `Esc`    | ファイル一覧へ戻る         |

### 期待動作

* `Ctrl+L` / `Alt+D` で現在のパスを選択状態にする
* そのまま入力すればパスを置き換えられる
* `Enter` で指定パスへ移動する
* `Esc` で編集をキャンセルし、ファイル一覧へフォーカスを戻す

---

## 4. フィルター操作

| 操作              | ショートカット  | 備考              |
| --------------- | -------- | --------------- |
| フィルターへフォーカス     | `Ctrl+F` | 現在ペインのフィルター欄へ移動 |
| フィルター確定         | `Enter`  | 既存仕様に従う         |
| フィルター解除 / 一覧へ戻る | `Esc`    | 既存仕様に従う         |

### 注意

フィルター入力中は、TextBox標準の操作を優先します。

例:

```text
Ctrl+C
Ctrl+X
Ctrl+V
Ctrl+A
Backspace
Delete
Left / Right
Home / End
IME入力
```

## 5. ファイル一覧操作

| 操作      | ショートカット  | 備考                           |
| ------- | -------- | ---------------------------- |
| 開く      | `Enter`  | ファイル実行 / フォルダ移動              |
| 名前変更    | `F2`     | 選択項目をリネーム                    |
| 削除      | `Delete` | ゴミ箱削除                        |
| コピー     | `Ctrl+C` | FileKakari内部コピー              |
| 切り取り    | `Ctrl+X` | FileKakari内部移動               |
| 貼り付け    | `Ctrl+V` | 内部Clipboard / 外部Clipboard両対応 |
| すべて選択   | `Ctrl+A` | 一覧内の項目を全選択                   |
| プレビュー切替 | `F3`     | プレビューペイン表示切替                 |
| 更新      | `F5`     | 現在フォルダを更新                    |

### 外部Clipboard貼り付け

Explorerなどでコピーしたファイル・フォルダも、FileKakari上で `Ctrl+V` または右クリックメニューから貼り付けできます。

入力先によって挙動が分かれます。

```text
ファイル一覧で Ctrl+V
    → ファイル貼り付け

TextBox内で Ctrl+V
    → テキスト貼り付け
```

## 6. メインタブ操作

メインタブは、FileKakariウィンドウ上部の作業単位タブです。

Workspaceタブ、通常フォルダタブ、設定タブなどを含みます。

| 操作             | ショートカット           | 備考           |
| -------------- | ----------------- | ------------ |
| 次のメインタブ        | `Ctrl+Tab`        | 右方向へ循環       |
| 前のメインタブ        | `Ctrl+Shift+Tab`  | 左方向へ循環       |
| 1〜8番目のメインタブへ移動 | `Ctrl+1`〜`Ctrl+8` | 左から数えたタブ     |
| 最後のメインタブへ移動    | `Ctrl+9`          | 最後のタブ        |
| 現在のメインタブを閉じる   | `Ctrl+W`          | 現在のメインタブを閉じる |

### 注意

* InternalPageタブもメインタブとして扱います
* 読み込み中のタブ切り替え制御は既存仕様に従います
* タブ切り替え時はViewState復元を維持します

---

## 7. サブタブ操作

サブタブは、Workspaceペイン内にあるフォルダタブです。

| 操作          | ショートカット         | 備考                |
| ----------- | --------------- | ----------------- |
| 次のサブタブ      | `Ctrl+PageDown` | アクティブペイン内で切り替え    |
| 前のサブタブ      | `Ctrl+PageUp`   | アクティブペイン内で切り替え    |
| 現在のサブタブを閉じる | `Ctrl+Shift+W`  | アクティブペインのサブタブを閉じる |

### 対象

* 単一ペインWorkspace
* 複数ペインWorkspace

通常フォルダでサブタブが存在しない場合は、何もしません。

### 保持される状態

サブタブ切り替え時は、既存のサブタブ選択経路を通し、以下を維持します。

* 選択状態
* スクロール位置
* アクティブペイン
* パスバー表示
* タイトルバー表示
* ListViewフォーカス

---

## 8. ペイン操作

ペインは、Workspace内のファイル一覧領域です。

| 操作       | ショートカット    | 備考         |
| -------- | ---------- | ---------- |
| 次のペインへ移動 | `F6`       | ペインを順方向に循環 |
| 前のペインへ移動 | `Shift+F6` | ペインを逆方向に循環 |

### 期待動作

ペイン移動時は以下を行います。

* アクティブペイン表示を更新する
* 対象ListViewへフォーカスする
* 既存選択を維持する
* 選択項目があればキーボードフォーカスを合わせる
* パスバーとタイトルバーを更新する
* サブタブは切り替えない
* 履歴は消費しない

### 今後の候補

方向指定のペイン移動は今後の候補です。

| 操作     | 候補ショートカット        |
| ------ | ---------------- |
| 左のペインへ | `Ctrl+Alt+Left`  |
| 右のペインへ | `Ctrl+Alt+Right` |
| 上のペインへ | `Ctrl+Alt+Up`    |
| 下のペインへ | `Ctrl+Alt+Down`  |

ただし、`Ctrl+Alt+方向キー` は他ツールやOS設定と競合する可能性があります。

## 9. TextBox / 入力欄で優先される操作

以下の入力欄では、標準のテキスト編集操作を優先します。

* パスバー
* フィルター欄
* リネームTextBox
* 設定画面内のTextBox
* ComboBox
* PasswordBox

入力欄では、以下をFileKakariのグローバルショートカットとして奪わないようにします。

```text
Ctrl+C
Ctrl+X
Ctrl+V
Ctrl+A
Backspace
Delete
Left
Right
Up
Down
Home
End
IME入力
```

### 例

```text
ファイル一覧で Backspace
    → 親フォルダへ移動

TextBox内で Backspace
    → 文字削除
```

```text
ファイル一覧で Ctrl+V
    → ファイル貼り付け

TextBox内で Ctrl+V
    → テキスト貼り付け
```

---

## 10. 入力処理の優先順位

キー入力は以下の順に扱います。

```text
1. リネームTextBox編集中
2. TextBox / ComboBoxなど標準入力コントロール
3. InternalPage
4. ファイル一覧 / Workspaceペイン操作
5. Window全体ショートカット
```

これにより、テキスト入力中の標準操作を壊さず、ファイル一覧にフォーカスがある場合だけFileKakariの操作を実行します。

---

## 11. 実装上の注意

### Alt系ショートカット

`Alt+Left` / `Alt+Right` / `Alt+Up` は、WPFの通常キーイベントに届かない場合があります。
そのため、FileKakariでは必要に応じて `ComponentDispatcher.ThreadPreprocessMessage` で `WM_SYSKEYDOWN` を捕捉します。

対象は以下に限定します。

```text
Alt+Left
Alt+Right
Alt+Up
```

一般ショートカットは通常の `Window_PreviewKeyDown` で処理します。

### 診断ログ

キー入力診断ログは通常時には出力しません。

* DEBUGビルドでは有効
* RELEASEビルドでは `FILEKAKARI_DIAG_LOG=1` の場合のみ有効

### イベント解除

`ComponentDispatcher.ThreadPreprocessMessage` を登録する場合は、Window終了時に解除し、二重登録を防止します。
