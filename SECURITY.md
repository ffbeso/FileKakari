# Security Policy

## セキュリティ脆弱性の報告方法
セキュリティに関する重大な問題や脆弱性を発見した場合は、**通常の公開 Issue に詳細を書き込まないでください。**

1. **プライベート脆弱性報告の利用**:
   * GitHub の Security Advisories 機能（Private vulnerability reporting）が利用可能な場合は、そちらから安全に報告を行ってください。
2. **通常バグの扱い**:
   * セキュリティ上のリスクを伴わない一般的なバグについては、通常通り公開 Issue へ投稿してください。

## 修正ポリシーと保証について
* FileKakari は初期プロトタイプ開発段階にあります。そのため、報告された脆弱性に対する修正の保証期間や即時対応の確約はいたしかねます。あらかじめご了承ください。
